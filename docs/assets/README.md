# docs/assets/

Static assets served by GitHub Pages at
`https://eriic-builds.github.io/wc26-bracket/assets/`.

## ⚠️ Needs one manual asset: `og-preview.png`

The dashboard's OpenGraph/Twitter tags (in `scripts/build_dashboard.py`) point at
`assets/og-preview.png` so shared links unfurl with a preview image. **That image
cannot be generated by the build** — a maintainer must add it once:

1. Open `docs/index.html` in a browser at ~1280px wide, **dark theme**, scrolled to
   show the hero + KPI row.
2. Screenshot and crop/resize to **exactly 1200×630** (`sips -g pixelWidth -g pixelHeight
   og-preview.png` to verify); keep it **under 300 KB**.
3. Save it here as `og-preview.png` and commit. No generator change is needed — the
   tags already reference it.
4. Also set the repo-level card: **Settings → General → Social preview → upload the
   same PNG** (this can't be done via the API/CLI).

Until the PNG lands, the `og:image` tag and the README preview embed resolve to a 404;
everything else (title, description, favicon, canonical) works immediately.
