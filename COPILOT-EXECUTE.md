## Global rules — read first, apply to every task

1. **One task = one branch = one PR.** Branch names: `agent/task-1-ci-tests`,
   `agent/task-2-fixtures`, `agent/task-3-json-data`, `agent/task-4-tz-observability`,
   `agent/task-5-og-preview`. Never combine tasks. Never start task N+1 until task N's PR is
   merged (task order is a dependency chain).
2. **`docs/index.html` is build output — never hand-edit it.** It is generated by
   `python scripts/build_dashboard.py`. Any change to page content/markup/CSS goes into the
   generator; then regenerate and commit BOTH files together. This rule has been violated
   before and caused a documented regression (see closed issue #2).
3. **The tournament is live.** The repo auto-commits to `main` up to 3×/day via
   `.github/workflows/sync-results.yml` (it rewrites `RES=`, `UPCOMING=`, `AUTO_HL=`,
   `REFRESHED=` inside `scripts/build_dashboard.py` and regenerates the HTML). **Rebase your
   branch onto latest `main` immediately before opening the PR and again before merge.** If
   you see conflicts in those data blocks, take `main`'s side for the data and re-apply your
   structural change.
4. **Python constraints:** Python 3.12, **standard library only** — do not add pip
   dependencies, `requirements.txt`, or type-checking configs. Match the existing code style
   (compact, commented for the "why", f-strings, `encoding="utf-8"` on every file open).
5. **Verify before you push.** Every task lists verification commands. Run them. If one fails,
   fix the code — do not weaken the check, delete a test, or mark a criterion "N/A".
6. **Do not touch:** `input/` (the user's source workbook and the 100 KB kit file),
   `docs/how-i-built-this.html`, `LICENSE`, the cron schedules in `sync-results.yml`
   (except where task 4 says so), and `OPUS-BUILD-fan-bracket-viewer.md` (a spec for a
   different repo).
7. **PR description format:** what changed, why (one paragraph), the verification commands you
   ran with their output summarized, and a checklist mirroring the plan's acceptance criteria
   with every box checked. If a criterion could not be met, STOP and report — do not merge.
8. **Network access:** the FIFA feed (`api.fifa.com`) is public and keyless. If your sandbox
   has no network, every task below still has an offline verification path — use it and note
   in the PR that live-feed verification is pending a maintainer run.
